# 이슈 29 위치 이력 보관 정책 정리

기준일: 2026-06-26

## 구현한 내용

- 위치 이력 보관 및 화면 노출 정책 초안을 `docs/operations/location-history-retention-policy.md`에 추가했다.
- 현재 구현 기준인 `companionSessions.sharedLocationHistory` 최근 10건 유지 정책을 데이터 계약 문서에 명시했다.
- 최신 위치 좌표, 마지막 갱신 시각, 실시간 위치 공유 상태 필드를 `CompanionSession` 데이터 계약에 보강했다.
- 보호자/환자, 매니저, 관리자 화면별 위치 정보 노출 범위를 문서화했다.
- 완료 후 원본 좌표 이력은 장기 분석 데이터로 보관하지 않는 기준을 문서화했다.
- 자동 정리, TTL, 감사 로그, 개인정보 삭제 요청 처리는 후속 구현 후보로 분리했다.

## 변경된 범위

- 운영 정책: 위치 이력 보관 및 노출 기준 추가
- 아키텍처 문서: `CompanionSession` 위치 필드와 최근 10건 유지 기준 보강
- 상태/기획/디자인 문서: 위치 이력 장기 보관 정책이 초안으로 정리됐다는 상태 반영
- 보고서: 이슈 29 작업 기록 추가

## 검증

- 문서 링크와 상대 경로를 확인했다.
- 코드 변경이 없는 문서 전용 작업이므로 Android 빌드와 단위 테스트는 실행하지 않았다.

## 남은 범위

- 완료/취소 세션의 원본 좌표를 자동 정리하는 운영 스크립트 또는 Cloud Function 구현 여부 결정.
- Firestore TTL 적용 가능성 검토.
- 위치 이력 정리 전후를 기록하는 관리자 감사 로그 설계.
- 개인정보 삭제 요청 처리 절차와 운영 도구 분리.
- 완료 후 원본 좌표 정리 시점은 기획/운영 리뷰에서 확정 필요.
