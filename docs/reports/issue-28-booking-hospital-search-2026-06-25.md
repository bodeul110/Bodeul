# 이슈 28 예약 병원/약국 검색 정책 정리

기준일: 2026-06-25

## 구현한 내용

- 예약 병원 선택은 카카오 로컬 REST API 병원 검색을 우선 사용한다.
- 카카오 REST API 키가 없거나 검색/통신이 실패하면 기존 `hospitalGuides` 기반 병원 목록으로 fallback한다.
- 카카오 검색 결과가 관리자 병원 가이드와 정확히 일치하면 해당 병원의 진료과 목록을 선택지로 보여준다.
- 카카오 검색 결과에 일치하는 진료과가 없거나 검색 결과가 없으면 병원명/진료과 직접 입력을 허용한다.
- 선택된 병원의 실좌표가 있으면 `appointmentRequests.hospitalLatitude`, `appointmentRequests.hospitalLongitude`에 저장한다.
- 좌표가 없는 직접 입력 또는 로컬 fallback 선택은 위도/경도를 `0.0`으로 저장한다.
- 관리자/보호자/예약 상세에서 병원 가이드가 없으면 공통 기본 병원 동행 스텝으로 대체한다.
- 매니저 지도 액션은 좌표가 있으면 카카오 지도 링크와 인근 약국 검색 링크를 사용하고, 좌표가 없으면 가능한 공식 병원 지도 fallback만 사용한다.
- Firebase 관리자/보호자/매니저 조회 경로도 `hospitalLatitude`, `hospitalLongitude`를 읽어 같은 예약 좌표를 사용한다.
- 카카오 검색 응답이 늦게 도착해도 최신 검색어 결과만 화면에 반영한다.

## 변경된 범위

- Android 예약 병원 선택 화면: 검색 결과 병합, 직접 입력 UI, 좌표 전달.
- Android 예약 저장 모델: 병원 위도/경도 저장 경로 유지 및 mock 저장소 좌표 누락 보완.
- Android 가이드 조합 저장소: Firebase/mock 상세 화면의 병원 가이드 누락 fallback.
- 문서: 데이터 계약, 07번 gap 체크리스트, 구현 PR 검증 시나리오.

## 검색/선택 UX 기준

- 검색어가 비어 있으면 관리자 가이드 기반 병원 목록만 보여준다.
- 카카오 REST API 키가 설정되어 있으면 카카오 병원 검색 결과를 먼저 보여준다.
- 카카오 병원명과 `hospitalGuides.hospitalName`이 정확히 일치하면 그 병원의 `departmentName` 목록을 연결한다.
- 카카오 결과에 연결 가능한 진료과가 없으면 해당 병원명을 유지한 채 진료과 직접 입력 다이얼로그를 연다.
- 검색 결과가 비어 있어도 병원/진료과 직접 입력 버튼으로 예약을 계속 진행할 수 있다.
- 병원명과 진료과는 모두 필수 입력이며, 비어 있으면 저장하지 않는다.

## 데이터 저장 기준

- 병원명: `appointmentRequests.hospitalName`에 저장한다.
- 진료과명: `appointmentRequests.departmentName`에 저장한다.
- 병원 좌표: `appointmentRequests.hospitalLatitude`, `appointmentRequests.hospitalLongitude`에 저장한다.
- 카카오 결과 선택: 카카오 결과의 위도/경도를 저장한다.
- 직접 입력 또는 좌표 없는 fallback: 위도/경도를 `0.0`으로 저장한다.
- 관리자 가이드 연결: `hospitalGuides.hospitalName`과 `hospitalGuides.departmentName` 조합으로 조회한다.
- 가이드 조회 실패: 저장된 예약 데이터는 유지하고 공통 기본 동행 스텝을 화면 모델에 사용한다.

## 구현 PR 검증 시나리오

1. 카카오 REST API 키가 있는 환경에서 병원명을 검색하면 `HP8` 병원 검색 결과가 노출되고, 관리자 가이드와 일치하는 병원 선택 시 진료과 선택 후 좌표가 예약 요청에 저장된다.
2. 카카오 REST API 키가 없는 환경에서 병원명을 검색하면 앱이 실패하지 않고 `hospitalGuides` 기반 결과만 노출되며, 선택한 예약 요청의 좌표는 `0.0`으로 저장된다.
3. 카카오 병원 결과는 있지만 관리자 가이드와 일치하는 진료과가 없으면 직접 입력 다이얼로그가 열리고, 입력한 진료과와 카카오 좌표가 예약 요청에 저장된다.
4. 검색 결과가 없으면 직접 입력 버튼으로 병원명/진료과를 입력해 예약을 완료할 수 있고, 좌표는 `0.0`으로 저장된다.
5. 예약 요청의 병원/진료과에 해당하는 `hospitalGuides` 문서가 없어도 매니저/보호자/예약 상세 화면은 공통 기본 병원 동행 스텝을 표시한다.
6. 매니저 가이드 화면에서 좌표가 있는 예약은 카카오 병원 지도 링크와 인근 약국 검색 링크를 제공하고, 좌표가 없는 예약은 좌표 기반 링크를 만들지 않는다.

## 수동 검사 방법

1. 카카오 REST API 키가 있는 환경에서 예약 생성 화면의 `병원/진료과 찾기`를 열고 병원명을 검색한다.
2. 검색 결과 병원을 선택하고 진료과를 선택한 뒤 예약 화면으로 돌아와 병원명과 진료과가 입력칸에 남는지 확인한다.
3. 같은 예약에서 만남 위치 선택 화면을 열었다가 돌아와 만남 위치 입력칸도 유지되는지 확인한다.
4. 검색 결과가 없거나 진료과가 없는 병원에서 `병원/진료과 직접 입력`을 눌러 병원명과 진료과를 입력하고 예약 화면에 반영되는지 확인한다.
5. 예약을 완료한 뒤 Firestore `appointmentRequests/{id}`에 `hospitalName`, `departmentName`, `hospitalLatitude`, `hospitalLongitude`가 저장됐는지 확인한다.
6. 카카오 REST API 키가 없는 환경에서 같은 검색을 반복해 앱이 실패하지 않고 관리자 가이드 기반 결과 또는 직접 입력으로 진행되는지 확인한다.
7. 좌표가 저장된 예약을 매니저 계정에서 열고 병원 지도/인근 약국 액션이 표시되는지 확인한다.

## 남은 범위

- 실제 운영 카카오 REST API 키와 네트워크 환경에서 검색 품질을 샘플 병원명으로 확인해야 한다.
- 병원명 exact match가 운영 데이터에서 충분하지 않으면 별도 alias/정규화 테이블을 추가해야 한다.
