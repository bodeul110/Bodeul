# MVP 범위

## 목표

보들의 첫 번째 사용 가능한 흐름은 아래와 같습니다.

환자 또는 보호자 신청 -> 매니저 매칭 -> 방문 전 알림 -> 동행 진행 공유 -> 보호자 리포트

## 사용자 유형

- 환자: 병원 방문 동행을 신청합니다.
- 보호자: 환자의 진행 상태와 리포트를 확인합니다.
- 동행 매니저: 환자와 함께 이동하고 병원 가이드를 수행합니다.
- 관리자: 병원 가이드, 예약, 사용자, 매칭 상태를 관리합니다.

## 핵심 기능

### 환자와 보호자

- 로그인 또는 기본 사용자 정보 입력
- 병원 방문 동행 요청 생성
- 병원명, 진료과, 예약 시간, 만남 장소, 보호자 연락처, 특이사항 입력
- 현재 요청 상태 확인
- 방문 전 안내 메시지 확인
- 동행 진행 상태와 최종 리포트 확인

### 동행 매니저

- 배정된 동행 세션 목록 확인
- 환자, 병원, 진료과, 만남 장소, 예약 시간 확인
- 접수, 검사, 대기, 진료, 수납, 귀가 단계 가이드 수행
- 세션 진행 상태 업데이트
- 보호자에게 위치 및 상태 공유
- 최종 리포트 작성

### 관리자

- 병원 및 진료과 가이드 데이터 관리
- 매칭 상태 관리
- 사용자와 매니저 프로필 관리
- 세션 리포트 검토

## 1차 릴리스 범위

포함:

- AI 매칭 이전의 수동 또는 간단한 규칙 기반 매칭
- 일부 병원과 진료과를 위한 기본 가이드 데이터
- 앱 상태 기반의 기본 상태 업데이트
- 실시간 지도 대신 수동 위치 또는 예시 좌표 기반 UI

제외:

- 결제 연동
- 완전한 AI 최적화
- 대규모 병원 가이드 데이터베이스
- 실시간 지도 고도화
- 대외 홍보용 웹사이트
