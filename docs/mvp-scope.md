# 기능 범위 정렬 문서

기준일: 2026-04-23

이 문서는 초기 축소 MVP 문서를 최신 기능설명서 기준으로 다시 정렬한 범위 문서다.

## 기준

- 최상위 기능 기준은 최신 `보들 플랫폼 기능설명서`를 따른다.
- 화면 정보 구조는 `docs/restructure-target-map.md`를 따른다.
- 실제 구현 완료 여부는 `docs/implementation-status.md`에서 관리한다.

## 제품 범위

### 1. 진입 및 계정 설정

- 첫화면
- 로그인 및 회원가입
- 접근 권한 안내
- 사용자 유형 선택
- 매니저 자격 인증

### 2. 서비스 요청 및 예약

- 환자 건강 프로필 등록
- 동행 서비스 예약 신청
- 병원 검색과 만남 위치 선택
- 결제 및 쿠폰

### 3. 매칭 및 홈 화면

- 매니저 메인 홈
- 환자/보호자 메인 홈
- 실시간 매칭 진행
- 매칭 확정 및 준비사항

### 4. 실시간 동행

- 실시간 위치 및 안심 채팅
- 실시간 진료 가이드
- 실시간 매니저 리포트
- 약국 동행 및 복약 안내

### 5. 서비스 종료 및 정산

- 최종 동행 완료 리포트
- 만족도 평가 및 후기
- 마이페이지 및 정산 관리
- 고객센터 및 긴급 신고

## 기술 기준

- 앱은 `Android 네이티브(Java/XML)`를 유지한다.
- 데이터 계층은 `Firebase Auth + Firestore + Cloud Functions`와 목업 저장소를 병행한다.
- 소셜 로그인, 지도, 결제, 알림, 푸시, 후속 운영 기능은 화면 구조와 도메인 모델을 먼저 맞추고 실제 서비스 연동은 후속 단계에서 고정한다.
- 실시간 매칭과 운영 자동화는 기능 범위에 포함하지만, 실제 추천 규칙과 발송 채널은 점진적으로 고도화한다.

## 현재 범위 해석

- 기능설명서에 있는 화면과 흐름은 문서 기준 범위에 포함한다.
- 외부 서비스 벤더 선택, 정밀 실내 내비게이션, 고도화된 AI 추천 엔진 같은 구현 세부는 후순위로 둔다.
- 따라서 `AI 매칭 UI/흐름`, `정산 관리`, `SOS`, `매니저 내 페이지`, `관리자 후속 운영`은 더 이상 범위 밖으로 보지 않는다.

## 후순위 항목

- iOS 동시 출시
- 특정 PG사나 알림톡 대행사에 종속된 최종 벤더 고정
- 병원 실내 정밀 내비게이션
- 추천 정확도 고도화를 위한 별도 AI/ML 파이프라인
