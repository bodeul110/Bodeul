# MVP 범위

기준일: 2026-04-15

## 목표

보들의 첫 출시 범위는 아래 한 줄로 정리한다.

환자/보호자 신청 -> 관리자 수동 매칭 -> D-7/D-3/D-1 사전 알림 -> 매니저 동행 진행 -> 보호자 리포트

## 기술 기준

- 앱은 현재 리포 기준으로 `Android 네이티브(Java)`를 유지한다.
- 백엔드는 `Firebase Auth + Firestore + Cloud Functions`를 기준으로 잡는다.
- 사전 알림은 `Cloud Functions`가 예약 데이터를 읽어 `appointmentReminderJobs` 작업 문서를 만들고, 실제 카카오 알림톡 연동은 그 다음 단계에서 붙인다.
- 현재 단계에서는 `Gemini CLI`, `FastAPI`, `실내 내비게이션`, `AI 매칭`을 MVP 범위에서 제외한다.

## MVP에 포함할 기능

### 환자 / 보호자

- 로그인, 회원가입, 프로필 보완
- 병원 동행 신청
- 신청 상태 조회
- 보호자 진행 현황 조회
- 최종 진료 리포트 조회

### 관리자

- 미배정 요청 조회
- 병원 가이드 등록
- 수동 매칭

### 매니저

- 배정된 세션 확인
- 병원 동행 가이드 진행
- 보호자 공유 메시지 입력
- 복약 메모 입력
- 최종 리포트 저장

### 서버

- 예약 생성 시 일정 정규화 필드 저장
- 매일 오전 9시 기준 D-7 / D-3 / D-1 알림 작업 생성
- 이후 카카오 알림톡 발송 함수로 연결할 수 있는 작업 큐 유지

## MVP에서 미루는 기능

- 환자가 매니저를 직접 선택하는 공개 매칭 화면
- 매니저 지원/수락 리스트
- 서류 제출 승인 플로우
- 정산, 계좌, 지급 관리
- GPS 기반 도착 판정
- 병원 실내 정밀 내비게이션
- 병명 기반 필요 서류 자동 추천
- AI 문구 생성, AI 매칭 알고리즘
- iOS 동시 출시를 위한 React Native 전환

## 서버 작업 목록

### 이미 있거나 이번 턴에 정리한 것

- 예약 문서 저장
- 예약 알림 작업 스케줄 함수

### 다음 단계

1. 대행사 스펙에 맞는 카카오 알림톡 payload 최종 조정
2. 요청 취소/변경 시 남아 있는 알림 작업을 `SKIPPED` 처리하는 정리 함수 추가
3. 신청 단계의 환자-보호자 연결 입력 보강

## 데이터 기준

### appointmentRequests

- `appointmentAt`: 화면 표시용 문자열
- `appointmentAtEpochMillis`: 서버 스케줄 계산용 정규화 값
- `appointmentDateKey`: `yyyy-MM-dd`

### appointmentReminderJobs

- `appointmentRequestId`
- `reminderStage`: `D7`, `D3`, `D1`
- `templateKey`
- `channel`
- `state`
- `reminderDateKey`
- `appointmentDateKey`
- `recipientUserIds`
- `messagePreview`
