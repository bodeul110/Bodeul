## 기능설명서 기준으로 계속 우선할 것

다음 항목은 피그마 표현보다 기능설명서의 실제 연동 범위를 먼저 따른다.

- GPS 동행 현황 / 실시간 위치 확인
- 안심 채팅
- 결제 / 쿠폰 / 정산
- 약국 동행 / 복약 안내
- 건강정보
- AI 음성 정리
- OCR 복약 비교

즉 피그마에 화면이 약하게 보이거나 없어도, 기능설명서에 있으면 계속 구현 대상이다.

## 최신 피그마에서 강하게 보이는 영역

### 1. 인증/공통

- 첫 화면
- 로그인
- 역할 선택
- 권한 안내

### 2. 환자/보호자 메인

- 상단 상태 카드
- 2x2 주요 액션 카드
- 하단 보조 카드
- 예약 완료 후 상태 변형 보드

### 3. 환자 진행/리포트

- 진행 상태 요약
- 병원 편 리포트
- 복약 변화 리포트
- 병원 내 지도

### 4. 매니저 홈/가이드

- 대기/진행 상단 카드
- 12단계 가이드 흐름
- 지도/메모/사진 분리 보드

### 5. 서류 등록/설정

- 파일 종류 구분
- 현재 상태
- 다음 액션

### 6. 개요 보드

- `Main.png`
- `Body.png`
- `보들 가이드.pdf`

이 자산들은 세부 기능보다는 카드 위계와 화면군 묶음을 판단하는 데 더 적합하다.

## 피그마에서 약하거나 독립 보드가 부족한 영역

- 실시간 GPS 좌표 추적
- 안심 채팅 상세
- 실제 결제/PG
- 약국 지도와 약품 구조화
- 건강정보 편집
- AI/OCR 후속 기능

이 영역들은 지금도 기능 기준이 우선이다.

## 현재 프로젝트 해석

### 기능 기준

- 기능 우선순위는 바뀌지 않는다.
- 완료/부분완료/후속설계 분류는 계속 `feature-spec-gap-checklist-2026-05-22.md`를 따른다.

### 디자인 기준

- 최신 디자인 기준은 Figma 원본 파일이다.
- `design_refs/local/` 아래 다운로드 산출물은 이전 기준 보조 세트 또는 임시 export/cache로만 유지한다.

### 적용 방식

- 기능 방향 관련 작업: 기능설명서 우선
- 카드 위계, 간격, 강조 순서, CTA 배치: 최신 피그마 우선

## 정리한 다음 작업 순서

1. 인증/공통 화면 polish
2. 환자/보호자 홈과 진행/리포트 화면 polish
3. 매니저 홈/가이드 polish
4. 서류 등록/내 페이지 polish
5. 설정/마이페이지 묶음 정리
