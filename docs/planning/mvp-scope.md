## 이번 단계 MVP에 포함

### 1. 진입 및 계정 설정

- 첫 화면
- 로그인/회원가입
- 접근 권한 안내 화면
- 사용자 유형 선택
- 매니저 자격 인증 업로드와 관리자 승인 대기

### 2. 서비스 요청 및 예약

- 환자 건강 프로필
- 병원/진료과 선택 기반 예약 신청
- 일정, 성별 선호, 요청 메모, 요금 요약
- 결제 상태 확인 단계와 쿠폰 입력

### 3. 매칭 및 홈 화면

- 환자/보호자 홈
- 매니저 홈
- 매칭 대기/확정 이후 기본 상태 흐름

### 4. 실시간 동행 중

- 진행 단계 타임라인
- 보호자 공유 메모
- 카카오 지도 기반 위치 공유와 안심 채팅
- 위치 요약/현장 메모
- 최종 리포트 연결

### 5. 서비스 종료 및 정산

- 최종 리포트
- 만족도/후기
- 후속 처리
- 문의/SOS
- 관리자 운영 화면과 관리자 웹 심사

## 부분 구현으로 허용하는 범위

- 카카오 로컬 API 키나 운영 정책이 없는 환경의 병원/위치 선택 대체 흐름
- 실제 PG 연동 전의 결제 상태 시뮬레이션
- 실시간 위치/안심 채팅의 운영 정책 고도화 전 기본 앱 연동
- 매니저 수익금/출금 전의 정산 상태 표시
- 관리자 알림톡/실제 외부 채널 전송 전의 앱/운영 기록 중심 처리

## 이번 단계에서 제외

- AI 음성 녹음 기반 진료 리포트 자동 생성
- OCR 기반 처방전/약봉투 인식 및 과거 처방 비교
- 건강정보 별도 프로필 영속 저장
- 초과 시간 자동 추가 결제
- 실운영용 카카오 알림톡/외부 메시지 채널 고도화

## 운영 원칙

- 최신 기능설명서의 항목을 문서에서 제거하지 않는다.
- 이번 단계에서 닫지 못한 항목은 `부분 구현` 또는 `후속 설계`로 남긴다.
- 기능 구현 판단은 항상 `implementation-status`와 함께 본다.
