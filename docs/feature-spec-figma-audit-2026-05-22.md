## 1. 기능설명서 기준 정리

### 1-1. 핵심 20개 기능

기능설명서는 아래 20개 항목을 제품 기준선으로 둔다.

1. 첫화면
2. 로그인 및 회원가입
3. 접근 권한 안내
4. 사용자 유형 선택
5. 매니저 자격 인증
6. 환자 건강 프로필 등록
7. 동행 서비스 예약 신청
8. 결제 및 쿠폰
9. 매니저 메인 홈
10. 환자/보호자 메인 홈
11. 실시간 매칭 진행 중
12. 매칭 확정 및 준비사항
13. 실시간 위치 및 안심 채팅
14. 실시간 진료 가이드
15. 실시간 매니저 리포트
16. 약국 동행 및 복약 안내
17. 최종 동행 완료 리포트
18. 만족도 평가 및 후기
19. 마이페이지 및 정산 관리
20. 고객센터 및 긴급 신고

### 1-2. 기능설명서에 명시된 추가 기획 메모

기능설명서는 20개 핵심 항목 외에 아래 후속 메모도 직접 포함한다.

- AI 음성인식/STT 기반 진료 리포트 자동 생성
- OCR/Vision 기반 처방전/약봉투 인식과 복약 비교
- 선결제와 초과 시간 추가 정산 규칙
- GPS 동행 현황과 실시간 위치 확인
- 동행 가이드 내 지도 fallback
- 방문/복약 이력 기반 건강정보 화면

### 1-3. GPS/지도 관련 명시

기능설명서에는 위치 기능이 간접 메모 수준이 아니라 직접 기능으로 들어 있다.

- 13번 항목: `실시간 위치 및 안심 채팅`
  - 구성: 지도 위 매니저 위치 표시, 전화 버튼
  - 기능: GPS 좌표 실시간 트래킹
- 16번 항목: `약국 동행 및 복약 안내`
  - 구성: 약국 위치 지도, 처방전 사진 등록, 복약 정보 입력
- 추가 메모:
  - `gps 동행 현황`
  - `실시간 위치 앱 확인`
  - `동행가이드에서의 지도`
  - `서울대 홈페이지 지도라도 볼 수 있게`

즉 기능설명서 기준으로는 `GPS/지도`가 명확한 요구사항이다.

### 2-2. 피그마에서 약하거나 보이지 않는 화면

최신 ZIP에는 아래 항목이 `독립 화면` 수준으로 선명하게 보이지 않는다.

- 결제/쿠폰 세부 흐름
- 매칭 애니메이션/AI 분석 상세
- 실시간 GPS 지도 트래킹
- 안심 채팅
- 약국 지도와 복약 입력 상세
- 건강정보 화면
- 초과 시간 자동 정산 상세

즉 피그마는 현재 `초기 진입`, `홈`, `서류 등록`, `권한`, `역할 선택` 쪽이 강하고,  
`실시간 동행`, `지도`, `결제`, `후속 AI/OCR 기능`은 기능설명서보다 표현이 약하다.

## 3. 기능설명서와 피그마의 관계

### 3-1. 둘 다 강하게 겹치는 영역

- 스플래시
- 로그인
- 역할 선택
- 권한 안내
- 환자/보호자 홈
- 매니저 홈
- 매니저 자격 인증

이 영역은 기능설명서와 피그마 방향이 크게 어긋나지 않는다.

### 3-2. 기능설명서가 더 강한 영역

- GPS 실시간 트래킹
- 지도 위 위치 표시
- 안심 채팅
- 약국 지도/복약 처리
- 결제/쿠폰/PG 연동
- AI 음성 정리
- OCR 복약 비교
- 건강정보 화면

이 영역은 현재 구현과 피그마 모두에서 `부분 구현 또는 후속 설계`로 보는 게 맞다.

### 3-3. 피그마가 더 강한 영역

- 홈 카드의 위계
- 상단 상태 카드 강조 방식
- 권한 안내 카드 표현
- 서류 등록 화면의 시각적 그룹화

이 영역은 기능 요구보다 `시각 polish 기준`으로 다루는 게 적절하다.

## 4. 현재 프로젝트 문서 기준에 반영할 해석

### 4-1. 기능 기준

- 최상위 기준은 계속 기능설명서다.
- `restructure-target-map.md`, `mvp-scope.md`, `architecture-draft.md`는 이 기준을 따라간다.

### 4-2. 디자인 기준

- 최신 피그마 ZIP은 `확정 명세`가 아니라 `화면 polish 참고본`으로 둔다.
- 기능설명서와 충돌하면 피그마보다 기능설명서를 우선한다.

### 4-3. GPS/지도 해석

- 기능설명서 기준: 명시 요구사항
- 피그마 기준: 일부 암시 또는 약한 표현
- 현재 구현 기준: 부분 구현

따라서 프로젝트 문서에서는 계속 아래처럼 유지하는 게 맞다.

- `GPS/지도/실시간 채팅`: 부분 구현
- `위치 요약/진행 상태/보호자 공유 메모`: 현재 구현 범위
- `실제 지도 트래킹/채팅/약국 지도`: 후속 구현 범위

## 5. 바로 이어서 볼 개선 포인트

1. 환자/보호자 홈 카드 위계 정리
2. 매니저 홈 카드군 정리
3. 권한 안내 화면 표현 단순화
4. 마이페이지/설정 묶음 정리
5. GPS/지도는 시각 확대보다 실제 연동 범위 확정이 우선

## 결론

- 기능설명서에는 `GPS 실시간 트래킹`, `지도`, `안심 채팅`, `약국 지도`, `건강정보`, `AI/OCR`가 명확히 들어 있다.
- 최신 피그마 ZIP은 `홈`, `권한`, `서류 등록`, `로그인/역할 선택` 쪽이 중심이고, 실시간 위치/지도 계열은 약하다.
- 그래서 문서 기준은 계속 `기능설명서 우선`, `피그마는 시각 참고`로 유지하는 게 맞다.
