## 기능설명서 기준으로 유지할 것

기능설명서는 계속 최상위 기준이다. 특히 아래 항목은 디자인보다 기능설명서를 먼저 따른다.

- GPS 동행 현황 / 실시간 위치 확인
- 안심 채팅
- 결제 / 쿠폰 / 정산
- 약국 동행 / 복약 안내
- 건강정보
- AI 음성 정리
- OCR 복약 비교

즉, 화면이 아직 약하게 보이더라도 기능설명서에 있으면 `후속 구현 대상`으로 유지한다.

## 최신 디자인 세트에서 강하게 보이는 영역

### 1. 인증/공통

- 스플래시
- 로그인
- 역할 선택
- 권한 안내

특징:
- 짧은 설명
- 큰 선택 카드
- 단일 주 액션

### 2. 환자/보호자 홈

- 상단 상태 카드
- 2x2 퀵 액션 카드
- 하단 보조 콘텐츠 카드

### 3. 환자 진행/상세

- 상태 요약
- 진행 단계
- 현장 메모와 안내 카드
- 주 액션 버튼

### 4. 매니저 홈/가이드

- 현재 대기/진행 상태를 강조한 상단 카드
- 2x2 액션 카드
- 단계 레일 + 포커스 카드 구조

### 5. 서류 등록

- 파일 종류 구분
- 현재 상태
- 다음 액션

## 디자인 세트에서 약하거나 독립 보드가 없는 영역

- 실시간 GPS 지도 추적
- 안심 채팅 상세
- 실제 결제/PG
- 약국 지도와 약품 구조화
- 건강정보 편집
- AI/OCR 후속 기능

이 영역은 지금도 기능 기준이 우선이고, 디자인은 보조 수준으로만 본다.

## 현재 프로젝트 해석

### 기능 기준

- 기능 우선순위는 바꾸지 않는다.
- `restructure-target-map.md`와 `feature-spec-gap-checklist-2026-05-22.md`의 기능 상태 분류는 유지한다.

### 디자인 기준

- 최신 디자인 세트는 `단일 ZIP 한 장`이 아니라 `분할 화면 세트` 기준으로 본다.
- 따라서 이후 화면 polish는 `design_refs/bodeul_split_screens/previews/contact_sheet.png`와 각 그룹별 PNG를 기준으로 잡는다.

### 적용 방식

- 기능 방향이 걸린 작업: 기능설명서 우선
- 카드 위계, 간격, 강조 단계, CTA 배치: 디자인 세트 우선

## 재정리한 다음 작업 순서

1. 인증/공통 화면 polish
2. 환자/보호자 홈과 진행 화면 polish
3. 매니저 홈/가이드 polish
4. 서류 등록/내 페이지 polish
5. 설정/마이페이지 묶음 정리

- 카카오 지도 SDK 내장 지도

## 결론

- 기능 기준은 계속 기능설명서다.
- 디자인 기준은 최신 분할 화면 세트다.
- 앞으로 문서와 계획은 `기능 우선순위는 유지`, `UI polish 우선순위만 디자인 기준으로 재정렬`하는 방식으로 관리한다.
