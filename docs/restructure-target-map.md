## 현재 리포와의 차이

- 현재 앱은 `스플래시 -> 역할 선택 -> 로그인`까지만 일부 유사하고, `권한 안내` 화면이 없다.
- 현재 예약 흐름은 병원/진료과/시간 입력 중심이며 `지도 검색`, `예상 비용`, `결제` 구조가 없다.
- 현재 매니저 홈은 배정 세션 중심이고 `실시간 AI 매칭 On/Off`, `주변 대기 환자`, `경력/문의` 중심 홈 구조와 차이가 있다.
- 현재 보호자 화면은 리포트 조회 중심이며 `환자/보호자 메인 홈`과 `실시간 대시보드` 구조가 약하다.
- 현재 동행 중 화면은 단계 진행과 메모 저장 중심이며 `위치 공유`, `현장 사진 피드`, `약국 동행` 구조가 부족하다.
- 현재 종료 단계에는 `만족도/후기`, `정산`, `긴급 신고` 구조가 없다.

## 이번 턴 반영 범위

- 객체지향 작업 원칙을 작업 규칙에 명시
- 진입 플로우를 `스플래시 -> 권한 안내 -> 유형 선택 -> 로그인` 구조로 재정렬
- 권한 안내 화면을 별도 객체와 활동으로 추가
- 유형 선택 화면을 피그마 카드 구조에 가깝게 재구성

## 이번 턴 추가 반영 범위

- 홈 데이터 조합, 안내 카드, 화면 바인딩을 별도 객체로 분리
- 보호자 역할은 `보호자 리포트`, 환자 역할은 `요청 현황` 중심으로 홈 문구와 액션을 분기
