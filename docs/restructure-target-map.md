# 화면 개편 목표 정리

기준일: 2026-05-22

## 기준 자료

- 원본 기준: [보들 플랫폼 기능설명서](보들_플랫폼_기능설명서.pdf)
- 현재 구현 기준: [현재 구현 상태](implementation-status.md)

이 문서는 최신 기능설명서를 현재 앱/웹 구조로 다시 풀어 쓴 문서다.  
기능설명서의 20개 항목과 추가 메모를 현재 프로젝트의 화면 축으로 매핑하고, 현재 구현 상태와 차이도 함께 정리한다.

## 목표 정보 구조

### 1. 진입 및 계정 설정

1. 첫 화면
2. 로그인 및 회원가입
3. 접근 권한 안내
4. 사용자 유형 선택
5. 매니저 자격 인증

### 2. 서비스 요청 및 예약

6. 환자 건강 프로필 등록
7. 동행 서비스 예약 신청
8. 결제 및 쿠폰

### 3. 매칭 및 홈 화면

9. 매니저 메인 홈
10. 환자/보호자 메인 홈
11. 실시간 매칭 진행 중
12. 매칭 확정 및 준비사항

### 4. 실시간 동행 중

13. 실시간 위치 및 안심 채팅
14. 실시간 진료 가이드
15. 실시간 매니저 리포트
16. 약국 동행 및 복약 안내

### 5. 서비스 종료 및 정산

17. 최종 동행 완료 리포트
18. 만족도 평가 및 후기
19. 마이페이지 및 정산 관리
20. 고객센터 및 긴급 신고

### 6. 추가 기획 메모

- AI 음성 녹음 기반 진료 리포트 정리
- OCR 기반 처방전/약봉투 인식
- 건강정보 화면
- 선결제, 초과 시간 정산, 상세 결제 정책
- 병원 지도 fallback

## 현재 구현과의 대응

### 구현 완료 또는 기본 흐름 확보

- 첫 화면, 로그인/회원가입, 사용자 유형 선택
- 매니저 자격 인증 업로드와 관리자 승인 대기 흐름
- 환자 건강 프로필, 예약 신청 기본 흐름, 쿠폰/요금 요약
- 환자/보호자 메인 홈, 매니저 메인 홈
- 매칭 확정 이후 예약 상세/진행 상태 흐름
- 매니저 진료 가이드, 보호자 리포트, 최종 동행 완료 리포트
- 후기, 후속 처리, 관리자 운영 화면, 관리자 웹 서류 심사
- 고객센터/문의와 긴급 신고의 기본 운영 흐름

### 부분 구현

- 접근 권한 안내
  - 안내 화면은 있다.
  - 실제 시스템 권한은 보안 정책상 최소화돼 있고, 현재 버전은 위치/카메라/저장소를 일괄 선요청하지 않는다.
- 병원 검색
  - 병원/진료과 선택 화면은 있다.
  - 지도 API 기반 검색과 병원 지도 시각화는 아직 부분 구현이다.
- 결제 및 쿠폰
  - 결제 확인 단계와 결제 상태 필드는 있다.
  - 실제 PG 연동과 선결제/초과요금 정책은 미완성이다.
- 실시간 매칭 진행 중
  - 대기/매칭 진행 화면과 상태 흐름은 있다.
  - AI 매칭 분석 표시와 위치 기반 실시간 푸시 연동은 축소 구현이다.
- 실시간 위치 및 안심 채팅
  - 진행 상태, 위치 요약, 보호자 공유 메모 흐름은 있다.
  - 실제 GPS 지도 트래킹과 채팅은 미완성이다.
- 실시간 매니저 리포트
  - 현장 메모와 리포트 구조는 있다.
  - 사진 피드와 이미지 기반 타임라인은 부분 구현이다.
- 약국 동행 및 복약 안내
  - 복약 메모와 후속 정보는 있다.
  - 처방전 사진/OCR, 약국 지도, 약국 완료 체크는 미완성이다.
- 마이페이지 및 정산 관리
  - 매니저/사용자 마이페이지와 후속 정산 상태는 있다.
  - 매니저 수익금 현황, 출금 신청, 계좌 정산은 축소 구현이다.

### 현재 미구현 또는 후속 설계 대상

- AI 음성 녹음 기반 리포트 자동 생성
- OCR 기반 복약 정보 비교
- 건강정보 화면
- 초과 시간 자동 정산 정책
- 관리자/보호자 카카오 알림톡 실운영 연동
- 병원 지도 fallback의 실제 운영 규칙

## 현재 문서 반영 원칙

- 최신 기능설명서의 20개 항목은 모두 이 문서의 기준 축으로 유지한다.
- 아직 구현되지 않은 항목도 문서에서 삭제하지 않고 `부분 구현` 또는 `후속 설계`로 명시한다.
- 실제 구현 상태와 차이는 [현재 구현 상태](implementation-status.md)에서 누적 관리한다.
