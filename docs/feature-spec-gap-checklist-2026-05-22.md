## 2026-06-19 기준 실행 순서

기준 원칙:

- 기능 우선순위는 `docs/local/보들_플랫폼_기능설명서.pdf`를 따른다.
- 화면 위계와 polish 우선순위는 `design_refs/local/보들 가이드.zip`, `design_refs/local/latest_figma_2026-06-19/`를 따른다.
- 즉, `무엇을 만들지`는 기능설명서가 정하고, `어떤 화면부터 다듬을지`는 최신 피그마가 정한다.

### 1. 인증/공통 화면 polish

- 첫 화면
- 로그인
- 역할 선택
- 권한 안내

이 단계는 최신 피그마와의 시각 차이를 줄이고, 내부 테스트 시작 화면의 완성도를 먼저 끌어올리는 목적이다.

### 2. 환자/보호자 홈과 진행/리포트 polish

- 홈 상단 상태 카드
- 예약 완료 후 상태 변형
- 실시간 진행 화면
- 병원 편/복약 변화 리포트
- 병원 내 지도 화면 위계

이 단계는 피그마에서 가장 화면 분리가 분명한 축이라, 실제 체감 완성도 상승 폭이 크다.

### 3. 매니저 홈/가이드 polish

- 매니저 홈 상단 상태 카드
- 동행가이드 12단 흐름 기준 단계 구획
- 지도/메모/사진 블록 분리

이 단계는 현재 기능은 충분하므로, 정보 묶음과 단계 가독성을 높이는 쪽이 우선이다.

### 4. 위치/GPS 기능 고도화

- 카카오 지도 SDK 내장 화면
- 좌표 기반 마커 표시
- 연속 위치 갱신 구조
- 위치 변경 이력

이 단계부터는 피그마보다 기능설명서 우선이다. 현재는 fallback과 1회 공유까지 구현돼 있으므로, 실제 위치 기능 완성도를 올리는 단계다.

### 5. 결제/정산 정책 고도화

- 결제 흐름 정리
- 정산 후속 상태 상세화
- 초과 시간/환불 정책 반영

현재는 UI와 후속 상태 구조가 있으므로, 실운영 규칙과 실제 처리 흐름을 붙이는 작업이 핵심이다.

### 6. 약국/복약 상세화

- 카카오 로컬 API 키 기반 실좌표 약국 지도 고도화
- OCR 복약 비교 후속 설계
- 자동 약품 비교 규칙 고도화

이 단계는 피그마의 `복약 변화` 보드와 기능설명서의 약국/복약 요구를 맞추는 작업이다.

### 7. 매칭/알림/운영 고도화

- 실시간 매칭 알림
- 운영자 후속 정책
- 보호자/매니저 알림 흐름

현재 구조는 이미 있으므로, 실시간성·운영성을 강화하는 단계로 본다.

### 8. 후속 설계 항목

- AI 음성 정리

이 단계는 제품 범위를 넓히는 작업이라, 앞선 핵심 흐름을 닫은 뒤 다루는 게 맞다.

## 2026-05-22 추가 메모

- 실시간 위치 및 안심 채팅 항목은 현재 실시간 위치 확인 화면, 카카오 지도 내장 화면, 병원/약국 마커, 세션 공용 안심 채팅, 채팅 푸시 알림, 읽음 상태, 이미지/PDF 첨부, 다건 첨부, 매니저 연속 GPS 좌표 공유와 위치 이력까지 구현된 상태다.
- GPS 동행 현황 / 실시간 위치 확인은 위치 요약과 1회 공유 단계를 넘어 연속 좌표 공유, 위치 이력, 병원/약국 근접 자동 알림, 최근 위치 이동 타임라인, 지도 경로형 위치 이력 시각화까지 올라왔고, 남은 범위는 세부 운영 정책 정리다.
- 공유 위치 좌표가 있을 때는 사용자/보호자 화면과 매니저 가이드 화면 모두 카카오맵 direct link를 우선 사용하도록 보강했다.
- 보호자 리포트와 실시간 위치 확인 화면에는 최근 위치 공유 시각이 추가됐다.
- 보호자/환자 실시간 위치 확인 화면과 보호자 리포트에 수동 새로고침 버튼을 추가해 최신 공유 위치/현장 메모를 다시 읽는 흐름을 보강했다.
