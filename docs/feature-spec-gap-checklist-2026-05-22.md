# 기능설명서 항목별 구현 체크리스트
기준일: 2026-05-22

이 문서는 [보들_플랫폼_기능설명서.pdf](보들_플랫폼_기능설명서.pdf) 기준으로 현재 구현 상태를 `완료`, `부분 완료`, `미구현`, `후속 설계`로 나눠 정리한 체크리스트다.

상태 기준:

- `완료`: 현재 데모와 내부 테스트 기준으로 기본 사용자 흐름이 닫혀 있음
- `부분 완료`: 화면과 저장 구조는 있으나 외부 연동, 실시간성, 운영 정책이 비어 있음
- `미구현`: 문서 요구는 있으나 현재 사용자 흐름으로 제공되지 않음
- `후속 설계`: 기능설명서 메모 수준이며 별도 제품 결정이나 구조 확장이 더 필요함

## 핵심 20개 항목

| 번호 | 항목 | 상태 | 현재 기준 |
|---|---|---|---|
| 01 | 첫 화면 | 완료 | 스플래시, 초기 로딩, 자동 로그인 체크 흐름이 있음 |
| 02 | 로그인 및 회원가입 | 부분 완료 | 이메일/패스워드 로그인과 가입은 가능하지만 네이버 로그인은 보안 이유로 비활성화 상태 |
| 03 | 접근 권한 안내 | 부분 완료 | 권한 안내 화면은 있으나 실제 권한 요청은 필요 시점 최소 권한 기준으로 처리 |
| 04 | 사용자 유형 선택 | 완료 | 매니저 / 환자·보호자 분기와 진입 흐름이 있음 |
| 05 | 매니저 자격 인증 | 완료 | 신분증, 요양보호사 자격증, 간호사 자격증, 범죄경력 조회서 업로드와 관리자 심사 흐름이 있음 |
| 06 | 환자 건강 프로필 등록 | 완료 | 건강 메모, 복약 정보, 이동 보조, 동행 유형 입력이 예약과 연결됨 |
| 07 | 동행 서비스 예약 요청 | 부분 완료 | 예약 생성, 병원/진료과/위치 선택 흐름은 있으나 지도 API 기반 검색은 없음 |
| 08 | 결제 및 쿠폰 | 부분 완료 | 결제 확인/요약 UI, 승인 상태, 후속 정산 문의 유형(초과 시간/환불/기타)은 있으나 실제 PG 연동과 자동 추가 정산은 없음 |
| 09 | 매니저 메인 홈 | 완료 | 실시간 매칭 대기, 서류/내 페이지/이력/문의 진입 흐름이 있음 |
| 10 | 환자/보호자 메인 홈 | 완료 | 요청 버튼, 진행 현황, 예약 요약, 주요 액션 카드 흐름이 있음 |
| 11 | 실시간 매칭 진행 중 | 부분 완료 | 매칭 대기와 상태 변화는 있으나 AI 분석, 위치 기반 알림은 없음 |
| 12 | 매칭 확정 및 준비사항 | 완료 | 매칭 정보, 준비사항, 예약 상세와 진행 화면 연결이 있음 |
| 13 | 실시간 위치 및 안심 채팅 | 부분 완료 | 위치 요약과 보호자 공유 메모는 있으나 실제 GPS 추적과 채팅은 없음 |
| 14 | 실시간 진료 가이드 | 완료 | 단계별 진행 상태, 매니저 가이드, 보호자 반영 흐름이 있음 |
| 15 | 실시간 매니저 리포트 | 부분 완료 | 현장 메모, 사진, 최종 리포트 구조는 있으나 실시간 미디어 피드와 자동화는 없음 |
| 16 | 약국 동행 및 복약 안내 | 부분 완료 | 복약 메모, 약국 진행 요약, 약국 단계 완료는 있으나 약국 지도/OCR/약품 구조화는 없음 |
| 17 | 최종 동행 완료 리포트 | 완료 | 진료 요약, 다음 방문, 최종 리포트 흐름이 있음 |
| 18 | 만족도 평가 및 후기 | 완료 | 후기, 별점, 도움 요청 평가 흐름이 있음 |
| 19 | 마이페이지 및 정산 관리 | 부분 완료 | 프로필, 이용 내역, 정산 후속 상태는 있으나 실제 출금 요청과 계좌 정산 정책은 없음 |
| 20 | 고객센터 및 긴급 신고 | 완료 | 문의하기, FAQ 성격의 진입, SOS/관리자 후속 처리 구조가 있음 |

## 추가 기획 메모

| 항목 | 상태 | 현재 기준 |
|---|---|---|
| AI 음성인식 기반 진료 리포트 자동 생성 | 후속 설계 | 문서와 메모만 있고 기능 없음 |
| OCR 기반 처방전 인식과 복약 비교 | 후속 설계 | 문서와 메모만 있고 기능 없음 |
| 건강정보 화면 | 부분 완료 | 환자 홈에서 최근/진행 중 예약 기준 건강정보 읽기 화면 제공, 별도 프로필 영속 저장은 없음 |
| 선결제 + 초과 시간 자동 정산 규칙 | 부분 완료 | 결제/정산 구조와 후속 정산 문의 유형 저장은 있으나 실제 운영 규칙과 자동 추가 결제는 없음 |
| GPS 동행 현황 / 실시간 위치 확인 | 부분 완료 | 위치 요약은 있으나 실제 지도 기반 실시간 추적은 없음 |
| 동행 가이드 지도 fallback | 부분 완료 | 외부 지도 앱/검색 기반 fallback은 있으나 지도 API 내장 화면은 없음 |

## 우선순위로 보면 큰 gap

### 1. 부분 완료지만 영향이 큰 항목

- 07. 예약 요청의 지도/검색 고도화
- 08. 실제 결제/PG 연동
- 11. 실시간 매칭 분석/알림
- 13. GPS 실시간 위치/안심 채팅
- 16. 약국 지도와 복약 상세 구조화
- 19. 실제 정산/출금 정책

### 2. 후속 설계로 분리해야 하는 항목

- AI 음성 정리
- OCR 복약 비교
- 자동 초과 정산 고도화

## 현재 판단

- 데모와 내부 테스트 기준으로는 핵심 사용자 흐름이 대부분 닫혀 있다.
- 기능설명서 기준 gap은 새 화면 부재보다 `실시간 기능`, `외부 서비스 연동`, `운영 정책` 쪽에 집중돼 있다.
- 다음 개발 우선순위는 새 화면 추가보다 `위치`, `결제`, `정산`, `약국 상세`의 실제 연동 범위를 어디까지 가져갈지 정하는 쪽이 맞다.
