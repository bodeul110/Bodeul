# 아키텍처 초안

## 확정 스택

보들은 Java, XML 레이아웃, Gradle 기반의 네이티브 Android 앱으로 구현합니다.

## 패키지 구조

아래 구조를 기본 방향으로 사용합니다.

```text
com.example.bodeul
  data           API 클라이언트, 저장소, 로컬 저장소, 목업 데이터
  domain.model   기능 간에 공유하는 도메인 모델
  ui             앱 진입 및 공통 화면
  ui.admin       관리자 가이드 및 매칭 관리
  ui.auth        스플래시, 역할 선택, 로그인
  ui.booking     환자/보호자 신청 흐름
  ui.common      공통 UI 보조 클래스
  ui.manager     동행 매니저 흐름
  ui.report      보호자 리포트 흐름
```

## 기능 경계

- 신청 흐름: 환자/보호자 신청 폼과 신청 상태 확인
- 매칭 흐름: 매니저 배정과 상태 변경
- 리마인드 흐름: 방문 전 체크리스트와 안내 메시지
- 매니저 가이드 흐름: 병원 가이드 단계와 진행 업데이트
- 리포트 흐름: 보호자용 진행 현황과 최종 리포트
- 관리자 흐름: 가이드 데이터 및 매칭 상태 관리

## 연동 방식

백엔드가 준비되기 전에는 메모리 기반 또는 목업 저장소로 UI 개발을 진행합니다.

백엔드가 준비되면 화면 로직은 유지하고 저장소 구현만 Firebase 또는 API 기반으로 교체합니다.
