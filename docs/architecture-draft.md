## 확정 스택

보들은 Java, XML 레이아웃, Gradle 기반의 네이티브 Android 앱으로 구현한다.

## 패키지 구조

아래 구조를 기본 방향으로 사용한다.

```text
com.example.bodeul
  data           Firebase 저장소, 목업 저장소, 서비스 로케이터
  domain.model   기능 간에 공유하는 도메인 모델
  ui             루트 화면
  ui.admin       관리자 운영, 후속 처리, 알림, 전달 기록
  ui.auth        스플래시, 권한 안내, 역할 선택, 로그인, 프로필 보완
  ui.booking     환자/보호자 예약, 상세, 완료, 후속 처리
  ui.common      공통 화면 보조 객체와 진행 모델
  ui.home        환자/보호자 메인 홈
  ui.manager     매니저 홈, 가이드, 과거 이력, 문의, 내 페이지
  ui.report      보호자 진행 현황 및 리포트
  util           공통 유틸리티
```

## 화면 계층 원칙

- 액티비티에는 인증 확인, 저장소 호출, 화면 전환 같은 흐름 제어만 둔다.
- 화면 조합은 `Coordinator` 계열 객체에서 담당한다.
- 뷰 반영은 `Binder` 계열 객체에서 담당한다.
- 문구와 상태 표현은 `PresentationFormatter` 계열 객체에서 담당한다.
- 데이터 접근은 `Repository` 계약과 구현체에서 담당한다.

## 기능 경계

- 진입 및 계정 설정: 스플래시, 권한 안내, 역할 선택, 로그인, 프로필 보완, 매니저 자격 인증
- 서비스 요청 및 예약: 건강 프로필, 예약 생성, 병원 선택, 위치 선택, 결제 승인, 완료 화면
- 매칭 및 홈: 환자/보호자 홈, 매니저 홈, 매칭 대기, 매칭 확정 후 준비사항
- 실시간 동행: 위치 공유, 진료 가이드, 현장 메모, 리포트, 약국/복약 안내
- 종료 및 운영: 후기, 정산, SOS, 매니저 과거 이력/문의/내 페이지, 관리자 운영 화면

## 연동 방식

- 백엔드가 준비되기 전에는 목업 저장소와 Firebase 저장소를 병행해 UI와 흐름을 먼저 맞춘다.
- 저장소 계약은 유지하고 구현체만 바꿔 실서비스 API로 전환할 수 있게 만든다.
- 최신 기능설명서에 있는 항목은 화면 구조와 도메인 모델에 먼저 반영하고, 외부 서비스 연동은 후속 작업으로 분리한다.
