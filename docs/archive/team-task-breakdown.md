# 팀 작업 분담

이 문서는 최신 기능설명서 기준으로 팀 작업을 나누기 위한 시작점이다.

## 5인 기준 권장 분담

### 1. 프로젝트 리드 및 UX

- 기능설명서와 실제 문서 체계 정렬
- Android 구조 일관성 유지
- 피그마/와이어프레임 기준 화면 흐름 관리
- 발표와 데모 시나리오 관리

### 2. 환자 및 보호자 앱

- 진입/권한/역할 선택 이후 사용자 메인 홈
- 건강 프로필, 예약, 완료, 상세, 후기/정산/SOS
- 보호자 진행 현황과 최종 리포트

### 3. 동행 매니저 앱

- 매니저 메인 홈
- 동행 가이드와 실시간 진행 입력
- 과거 이력, 문의하기, 내 페이지
- 서류/일정/리포트 흐름

### 4. 관리자, 데이터, 백엔드

- 관리자 운영 화면
- 예약/후속/문의/알림/전달 기록 데이터 구조
- Firebase 및 후속 API 계약
- 가이드/정산/긴급 이슈 운영 로직

### 5. 통합 및 QA

- 빌드와 실행 가이드 관리
- 샘플 데이터 유지
- 전체 화면 흐름 점검
- 문서와 구현 차이 추적
- 발표용 스크린샷과 데모 체크리스트 준비

## 통합 데모 시나리오

1. 사용자가 권한 안내와 역할 선택을 거쳐 로그인한다.
2. 환자 또는 보호자가 건강 프로필과 예약 정보를 입력한다.
3. 매칭/확정 이후 보호자와 환자 홈에서 진행 상태를 확인한다.
4. 매니저가 실시간 가이드, 현장 메모, 리포트를 입력한다.
5. 완료 이후 후기/정산/SOS와 관리자 후속 운영 흐름까지 확인한다.

## 작업 규칙

- 기능설명서와 어긋나는 문서는 최신 기준으로 함께 갱신한다.
- 기능 작업 중 관련 없는 대규모 리팩터링은 하지 않는다.
- 화면 작업은 기능별 폴더와 역할별 객체로 분리한다.
- API 계약이 고정되기 전까지 목업/Firebase 저장소를 병행한다.
- 담당자가 바뀌면 이 문서를 업데이트한다.
- 병합 전에 반드시 `assembleDebug`를 확인한다.
