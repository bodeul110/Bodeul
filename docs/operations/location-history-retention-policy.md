# 위치 이력 보관 및 노출 정책

기준일: 2026-07-18

## 목적

실시간 위치 공유는 동행 중 보호자 안심 확인과 운영 대응을 돕기 위한 기능이다. 위치 좌표는 민감한 개인정보에 해당하므로 MVP 단계에서는 원본 위치 이력을 장기 보관하지 않고, 진행 중 동행에 필요한 최소 범위만 유지한다.

## 현재 구현 기준

- 위치 이력은 `companionSessions.sharedLocationHistory` 배열에 저장한다.
- 세션 문서에는 최신 좌표 `sharedLatitude`, `sharedLongitude`, 마지막 갱신 시각 `sharedLocationUpdatedAtMillis`를 함께 저장한다.
- 연속 위치 공유 상태는 `liveLocationSharingActive`, `liveLocationSharingStartedAt`로 관리한다.
- 앱 도메인 모델은 `CompanionSession.MAX_SHARED_LOCATION_HISTORY = 10` 기준으로 최근 10건만 유지한다.
- 위치 공유가 종료되면 `liveLocationSharingActive`는 `false`로 바꾸고 공유 시작 시각은 제거한다.

## 보관 정책

| 데이터 | 보관 기준 | 목적 |
| --- | --- | --- |
| 최신 좌표 | 진행 중 세션 문서에 유지 | 보호자/환자 실시간 위치 확인, 매니저 현장 확인 |
| 위치 이력 배열 | 세션당 최근 10건만 유지 | 최근 이동 흐름 표시, 지도 경로 표시 |
| 위치 요약 텍스트 | 세션 문서에 유지 | 좌표 없이도 현장 상황을 설명 |
| 완료·취소 후 정밀 좌표 | 24시간 이내 삭제 | 개인정보 최소 보관, 지연 처리 재시도 |
| 수집·이용·제공사실 확인자료 | 6개월 | 좌표 없이 주체·목적·시각·제공 대상만 감사 기록 |
| 운영 리포트 | 좌표 원본 대신 요약 문구 중심으로 유지 | CS, 품질 점검, 분쟁 대응 |

MVP 기준으로 원본 좌표는 서비스 진행 중 확인 목적에 한정한다. 완료 또는 취소 시 `expires_at`을 종료 시각의 24시간 뒤로 설정하고 정리 job이 최신 좌표와 최근 10건을 삭제한다. 정리 지연 중에도 완료된 세션의 원본 좌표를 일반 화면에 노출하지 않으며 장기 분석 데이터로 사용하지 않는다.

## 화면 노출 기준

### 보호자/환자

- 진행 중 세션에서는 최신 위치, 최근 위치 이력, 마지막 갱신 시각을 표시한다.
- 완료 또는 취소된 세션에서는 위치 원본 이력보다 최종 리포트와 위치 요약을 우선 표시한다.
- 위치 정보가 없거나 공유가 중지된 경우에는 대기/공유 중지 상태를 명확히 표시한다.

### 매니저

- 본인이 배정된 진행 중 세션에서만 위치 공유 시작/중지와 위치 갱신을 수행한다.
- 최근 위치 이력은 현장 진행 확인과 보호자 공유 상태 점검용으로만 표시한다.
- 완료 또는 취소 후에는 위치 공유를 중지하고 추가 좌표 갱신을 하지 않는다.

### 관리자

- 관리자 화면은 운영 대응을 위해 최신 위치 상태, 위치 요약, 최근 이력을 제한적으로 확인할 수 있다.
- 장기 보관 목적의 전체 이동 경로 조회 화면은 MVP 범위에 포함하지 않는다.
- 분쟁 대응이 필요한 경우에도 원본 좌표 장기 보관보다는 세션 요약, 최종 리포트, 감사 로그를 우선 사용한다.

## 개인정보 및 운영 기준

- 위치 좌표는 목적 외 재사용을 금지한다.
- 보호자/환자에게 보이는 문구는 위치 공유 상태와 마지막 갱신 시각을 오해 없이 표현해야 한다.
- 위치 공유 중에는 배터리, 네트워크, 권한 상태에 따라 갱신이 지연될 수 있음을 UI에서 안내한다.
- 민감 위치 원본을 별도 리포트나 분석 데이터로 장기 저장하지 않는다.
- 개인정보 삭제 요청이 들어오면 해당 사용자와 연결된 진행 종료 세션의 위치 원본 삭제 범위를 우선 검토한다.
- 분쟁 또는 법적 의무에 따른 예외 보관은 대상 세션, 근거, 승인자와 종료일을 기록한 legal hold로만 허용한다.

## 후속 구현 후보

- 완료/취소 세션의 `sharedLocationHistory`, `sharedLatitude`, `sharedLongitude`를 24시간 뒤 정리하는 전환 기간 job.
- PostgreSQL 전환 뒤 `expires_at` 기반 일일 정리 job.
- 위치 이력 정리 전후를 기록하는 관리자 감사 로그.
- 개인정보 삭제 요청 처리 절차와 운영 도구.
- 위치 업데이트 빈도와 비용을 점검하는 운영 리포트.

## 확정 기준

- 정밀 좌표 삭제: 세션 완료·취소 후 24시간
- 위치정보 수집·이용·제공사실 확인자료: 6개월
- 장기 운영 리포트: 정밀 좌표와 전체 경로를 제외한 병원·행정구 또는 상태 요약만 허용
- 기술 기본 legal hold: 최대 180일, 더 긴 보관은 별도 법적 근거 필요

상세 공통 기준은 [데이터 보관 및 파기 정책](data-retention-policy.md)을 따른다.
