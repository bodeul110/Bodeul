# 팀 작업 분담

이 문서는 팀 작업을 나누기 위한 시작점입니다.

## 5인 기준 권장 분담

### 1. 프로젝트 리드 및 UX

- Java Android 구조 일관성 유지
- MVP 화면 흐름 정의
- Figma 또는 간단한 와이어프레임 작성
- 사업 계획과 요구사항 정렬
- 최종 발표와 데모 시나리오 관리

### 2. 환자 및 보호자 앱

- 신청 생성 화면 구현
- 신청 상태 확인 화면 구현
- 보호자 리포트 화면 구현
- 먼저 목업 데이터에 연결하고 이후 API 연동

### 3. 동행 매니저 앱

- 배정 세션 목록 구현
- 세션 상세 화면 구현
- 병원 가이드 단계 화면 구현
- 진행 상태 업데이트와 완료 처리 구현

### 4. 백엔드, 데이터, 관리자

- 데이터베이스 스키마 정의
- API 계약 정의
- 목업 서버 또는 실제 백엔드 구축
- 관리자 가이드 데이터 관리 화면 또는 시드 도구 구현

### 5. 통합 및 QA

- 빌드와 실행 가이드 관리
- 샘플 데이터 유지
- 전체 데모 시나리오 테스트
- 버그와 누락 흐름 추적
- 발표용 스크린샷과 데모 체크리스트 준비

## 첫 데모 시나리오

1. 환자 또는 보호자가 병원 방문을 신청합니다.
2. 관리자 또는 목업 매칭이 매니저를 배정합니다.
3. 매니저가 배정된 세션을 엽니다.
4. 매니저가 가이드 단계에 따라 진행 상태를 업데이트합니다.
5. 보호자가 현재 진행 상황과 최종 리포트를 확인합니다.

## 작업 규칙

- 기능 작업 중 관련 없는 리팩터링은 하지 않습니다.
- 화면 작업은 기능별 폴더로 분리합니다.
- API 계약이 안정화되기 전까지 목업 데이터를 사용합니다.
- 담당자가 바뀌면 이 문서를 업데이트합니다.
- 병합 전에 반드시 앱 빌드를 확인합니다.
